# ai-todo v0.7.0 版本规划 — 小程序来源可见与列表分组

日期：2026-06-08  
状态：**候选**；建议在 **v0.6.2（MCP）** 之后或并行筹备，按内测反馈择期发布。  
发布目标：让微信用户**在列表层**看到 Agent/外部系统创建的提醒，并改善待办信息层次；**无 API 契约变更**（复用 v0.6.0 字段）。

## 版本定位

| 组件              | 目标 L1               | Git tag | 是否变更       |
| ----------------- | --------------------- | ------- | -------------- |
| Git `release_tag` | `v0.7.0`              | 同上    | 小程序体验火车 |
| API               | `0.3.0`               | 同上    | **否**         |
| CLI               | `0.5.0`               | 同上    | **否**         |
| 微信小程序        | `0.6.0` → **`0.7.0`** | 同上    | **是**         |

前置：v0.6.0 已部署；用户可能已通过 CLI/Agent 写入带 `source` 的提醒。

---

## 背景

v0.6.0 仅在**提醒编辑页**只读展示来源；列表页无标识。v0.5.4 起 pending 已全量按 `due_at` 排序，但无「逾期 / 今天 / 未来」分组头。两项均为 v0.6.1 明确后置。

---

## P0：提醒列表来源角标

- 行内 subtle 标签：`email` →「邮件」、`jira` →「工单」、未知 source 显示 raw 值。
- 点击行进入编辑页查看 `sourceMeta`（已有）。
- 映射表：`lib/source-labels.ts`（可后续改 i18n）。

## P0：列表分组（逾期 / 今天 / 未来）

- 数据：`GET /v1/reminders?status=pending&sort=due_at`（已有）。
- UI：三段 `group-header`；无 `due_at` 仍排最后。
- 「今天」边界与 `/v1/today`、账户时区一致（v0.2.2 基线）。

## P1（可选）

- 关于页补充一句：「部分提醒由电脑端 Agent 创建，来源见编辑页」。
- 分享卡片/品牌 icon（若设计稿同期定稿，可并入本批；否则仍后置）。

---

## 非目标

- 触达可观测 / 运维看板
- 来源字段编辑（仍只读）
- API / MCP 变更
- 星标、撤销删除
- 平台 username / 建联
