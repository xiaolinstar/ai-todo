# v0.6.0 Release Notes

发布目标：结构化待办来源字段，为 Agent、邮件、微信消息、工单等外部系统接入提供服务端反查能力。

## 组件版本

| 组件              | 版本     | 说明                                              |
| ----------------- | -------- | ------------------------------------------------- |
| Git `release_tag` | `v0.6.0` | 待发布 tag                                        |
| API               | `0.3.0`  | Reminder 来源字段、唯一索引、lookup、source list  |
| CLI               | `0.5.0`  | `--source` / `--external-id` 创建、查询与快捷操作 |
| 微信小程序        | `0.6.0`  | 提醒编辑页只读展示来源，不提供编辑入口            |

## 主要变化

- `reminders` 增加 `source`、`external_id`、`source_meta`。
- 服务端新增唯一索引：`user_id + source + external_id`，仅约束非空来源外键。
- `POST /v1/reminders` 支持来源字段；重复来源命中已有 reminder，返回 `created=false`。
- `GET /v1/reminders/lookup?source=...&externalId=...` 支持来源反查。
- `GET /v1/reminders?source=...` 支持来源分桶列表。
- CLI 支持：
  - `ai-todo reminder create --source email --external-id ...`
  - `ai-todo reminder find --source email --external-id ...`
  - `ai-todo reminder list --source email`
  - `done/update/reschedule/delete --source ... --external-id ...`

## 设计说明

`source` / `external_id` 是业务来源，不等同于 `x-client-source`。前者描述待办来自邮件、微信消息、工单等；后者描述调用端是 CLI、小程序或 Agent。

`source` 暂不做硬枚举，避免每接入一个外部系统就修改 API。服务端只限制长度和字符集。
