# ai-todo v0.4.3 发布说明

生产 API 正式域名切换为 **xingxiaolin.cn**；API L1 bump 至 `0.2.2`。CLI / 小程序 L1 仍为 `0.4.0`。

## 组件版本

| 组件              | 版本     | 本批是否 bump              |
| ----------------- | -------- | -------------------------- |
| Git `release_tag` | `v0.4.3` | —                          |
| API               | `0.2.2`  | 是                         |
| CLI               | `0.4.0`  | 否                         |
| 微信小程序        | `0.4.0`  | 否（**需重新上传并提审**） |

详见 [api-0.2.2.md](./api-0.2.2.md)、[compatibility.md](./compatibility.md)。

## 变更摘要

- 文档、CLI 示例、API 根页、`cli_guidance.py` 默认 URL → `https://xingxiaolin.cn`
- 小程序 `lib/config.ts` 生产基址 → `xingxiaolin.cn`
- `wodi.games` 网关可过渡保留，供旧版体验版；新包与文档以 `xingxiaolin.cn` 为准

本 tag 亦包含 v0.4.2 规划内容（API `0.2.1`：CLI onboarding 文案与 `@xiaolinstar/ai-todo-cli` 对齐），未单独打 `v0.4.2` tag。

## 运维

1. GitHub Actions CD：`release_tag=v0.4.3`
2. Secret `CD_PUBLIC_API_URL` → `https://xingxiaolin.cn`（若尚未更新）
3. 微信公众平台 **request 合法域名** 添加 `xingxiaolin.cn`
4. 本地 CLI：`~/.ai-todo/settings.json` 中 `url` 改为 `https://xingxiaolin.cn`

## 验证

```bash
curl -sS https://xingxiaolin.cn/v1/health | jq '.data.apiVersion'
# 期望: "0.2.2"
```

## 小程序提审

- 微信上传版本号：**0.4.0**
- 确保已勾选合法域名、上传含新 `PRODUCTION_API_URL` 的代码包
