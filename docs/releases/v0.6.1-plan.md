# ai-todo v0.6.1 版本规划 — Agent 工具链闭环

日期：2026-06-08  
发布目标：在 **v0.6.0** 已提供来源字段 API/CLI 的前提下，把 **Agent 可调用的工具描述、Skill、文档与幂等工作流** 补齐；**不**做触达可观测，**不**要求重传微信小程序。

## 版本定位

| 组件 | 目标 L1 | Git tag | 是否变更 |
|------|---------|---------|----------|
| Git `release_tag` | `v0.6.1` | 同上 | 发布火车标识 |
| API | `0.3.0` | 同上 | **否** — 契约已在 v0.6.0 就绪 |
| CLI | `0.5.0` | 同上 | **否**（或仅 patch 文档链接时不 bump） |
| 微信小程序 | `0.6.0` | 同上 | **否** — 本批无需上传/提审 |
| `@ai-todo/agent-protocol` | `0.4.0` | 随仓 | **是** — 工具目录与 guidelines |
| `skills/ai-todo` | — | 随仓 | **是** — Cursor/OpenClaw Skill 正文 |

前置：生产已部署 **v0.6.0**（`source` / `lookup` / CLI 来源快捷操作可用）。

### 为何是 v0.6.1 而不是 v0.7.0

- v0.6.0 交付了**数据面**（库表、API、CLI）；本批交付**Agent 接入面**（工具规格、Skill、文档），是同一主题的收尾。
- 无 API 契约变更、无小程序 UI，不符合「跨组件 minor bump」的 v0.7.0 火车习惯。
- 触达可观测、品牌 UI、MCP 独立进程等仍后置到未来版本。

## P0：工具目录（agent-protocol）

在 [packages/agent-protocol/src/tools.ts](../../packages/agent-protocol/src/tools.ts) 新增/修订：

### 新增工具

| 工具名 | CLI 命令 | 典型场景 |
|--------|----------|----------|
| `ai_todo_reminder_find` | `reminder find --source <s> --external-id <id>` | 邮件 Agent 处理前先查是否已建提醒 |
| `ai_todo_reminder_create_sourced` | `reminder create … --source <s> --external-id <id> [--source-meta '<json>']` | 工单/邮件首次入库 |
| `ai_todo_reminder_list_by_source` | `reminder list --source <s> [--status pending]` | 按来源浏览（如 `email` 桶） |
| `ai_todo_reminder_update_by_source` | `reminder update --source <s> --external-id <id> [--title …] [--due …]` | 外部 ID 改期/改标题 |
| `ai_todo_reminder_complete_by_source` | `reminder done --source <s> --external-id <id>` | 外部系统标记完成 |
| `ai_todo_reminder_delete_by_source` | `reminder delete --source <s> --external-id <id>` | 外部系统撤销 |

### 修订既有工具

| 工具名 | 修订 |
|--------|------|
| `ai_todo_reminder_create` | `notes` 中注明：有稳定外部键时优先用 `ai_todo_reminder_create_sourced` |

### Guidelines 增补（`AI_TODO_AGENT_GUIDELINES`）

1. **`source` / `external_id` 是业务来源键**（`email`、`wechat`、`jira`…），不等于 HTTP 头 `x-client-source`（`cli` / `miniapp` / `agent`）。
2. **幂等写路径**：`find` → 若存在则 `update` / `done` / `delete`；不存在再 `create`（或依赖 API `created: false` 判断）。
3. **`source_meta`**：仅 JSON 展示/审计（如邮件 `subject`、`messageId`），服务端不解析 NL。
4. **`source` 不硬枚举**：新接入系统直接用新字符串，无需改 API。
5. **写操作带 `--idempotency-key`**（或 HTTP `Idempotency-Key`）防 Agent 重试重复写。

### 可选：工具 JSON 导出

新增脚本或 `agent-protocol` 导出函数，生成 `dist/agent-tools.json`，供 MCP 生成器或文档站消费（纯构建产物，无运行时依赖）。

## 推荐 Agent 工作流（本版本要教会 Agent 的）

### 邮件 → 提醒（幂等）

```text
1. Agent 从邮件解析：title、due、Message-ID
2. ai-todo reminder find --source email --external-id "<Message-ID>" --json
3. 若 ok 且 data.reminder 存在 → update/done 按用户意图
4. 否则 → reminder create --source email --external-id "..." \
     --source-meta '{"subject":"...","from":"..."}' --title "..." --due "..." --json
5. 检查 data.created：false 表示并发下已存在，改走 find 结果
```

### 工单关闭 → 提醒完成

```text
1. ai-todo reminder find --source jira --external-id "PROJ-123" --json
2. ai-todo reminder done --source jira --external-id "PROJ-123" --json
```

### 按来源批量查看

```text
ai-todo reminder list --source email --status pending --json
```
