# ai-todo v0.5.0 版本规划 — 微信订阅消息触达 + 小程序分享

日期：2026-06-06

## 背景

服务端自 v0.1.0 起已具备：订阅授权记录、delivery 队列、worker 进程、微信 send API。生产 compose 中 worker 在 `notifications` profile 下，**默认不启动**；真实模板 ID 与 openid 联调尚未作为发布门槛。

- [ ] 已有测试扩展：改期后 delivery 时间跟随

## 验收

### 本地 /  staging

```bash
cd apps/api && docker compose --env-file .env.local --profile notifications up -d
pnpm test:api
pnpm check:wechat
```

1. 创建带截止时间的提醒 → 接受订阅 → `GET /v1/notifications/status` 为 `pending`
2. 将 `dueAt` 改到 1 分钟后 → delivery `scheduled_at` 更新
3. worker 扫描 → 微信收到订阅消息（需真实 openid + 模板 ID）
4. 点击消息 → 小程序打开对应提醒编辑页

### 生产

1. 部署后 `docker ps` 可见 `worker` 容器
2. 内测账号完整走通创建 → 收消息 → 点击落地
3. 任意 Tab 页右上角可分享给好友，卡片标题与路径正确
