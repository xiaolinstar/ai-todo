# ai-todo v0.2.1 版本规划

日期：2026-06-03  
发布目标：微信小程序体验版 / 填实 0.2.0 占位能力（无新隐私类目）

## 版本定位

`0.2.1` 在 **`v0.2.0`（设置枢纽）** 之后，聚焦：

1. **统一「使用偏好」**：提醒、日历、通讯录三类偏好合并为同一设置类（一个入口、一页分区），与「通知设置」（渠道）明确分工。
2. **账号与安全**文案页、**提醒列表**信息层次等小步体验。
3. **不**做时区/username PATCH、按用户名加人（留给 `v0.2.2`）；**不**做星标/撤销删除/MCP（留给 `v0.3.0`）。

| 组件       | 目标 L1                 | Git tag  |
| ---------- | ----------------------- | -------- |
| 微信小程序 | `0.2.1`                 | `v0.2.1` |
| API        | `0.1.3`（无契约变更）   | 同上     |
| CLI        | `>=0.1.4`（无强制变更） | 同上     |

## 设置信息架构调整（相对 0.2.0）

### 分工原则

| 类型         | 含义                                                                    | 入口                                    |
| ------------ | ----------------------------------------------------------------------- | --------------------------------------- |
| **通知设置** | 渠道与送达：微信总开关、订阅模板状态、免打扰时段、最近发送摘要          | 独立页 `notification-settings`          |
| **使用偏好** | 使用产品时的**默认行为与展示**：提醒默认值、日历默认值、通讯录展示/排序 | **单一入口** → `settings-content-prefs` |

提醒偏好、日历偏好、通讯录偏好**不再**各占菜单一行、三个占位页；合并为「使用偏好」下的三个 **inset 分组**（同一滚动页）。

### 「我的」菜单目标结构（0.2.1）

| 分组         | 入口                   | 0.2.1 状态                       |
| ------------ | ---------------------- | -------------------------------- |
| 账户         | 个人资料               | 保持                             |
| 账户         | 时区                   | 仍占位 → `0.2.2`                 |
| 账户         | 账号与安全             | **实装文案页**                   |
| 提醒与通知   | 通知设置               | 收敛为渠道；迁出「新建默认通知」 |
| **使用偏好** | **提醒、日历与通讯录** | **新页实装**                     |
| 扩展         | CLI / Agent 接入       | 保持                             |
| 关于         | 关于 ai-todo           | 保持                             |
| 开发         | 开发者选项             | 保持 develop                     |

删除原分组「日历与通讯录」及其中两个占位项；删除「提醒与通知」下的「提醒偏好」单行。

配置实现：改 `lib/settings-menu.ts`；废弃 placeholder slot `reminders` / `calendar` / `contacts` 的菜单跳转（可保留 `settings-placeholders.ts` 条目仅供历史链接兼容，或删除并在规划中注明）。

## 功能范围

### P0 — 必须交付

#### 1. 使用偏好页 `pages/settings-content-prefs/`

单页三块，共用 `settings-page` 样式与设计令牌。

**（1）提醒**

| 项                       | 数据来源                                                        | 行为                                                         |
| ------------------------ | --------------------------------------------------------------- | ------------------------------------------------------------ |
| 新建提醒默认开启微信通知 | `PUT/GET /v1/notifications/settings` → `defaultReminderEnabled` | 开关；保存时调用既有 API（与通知总开关联动：总开关关则禁用） |
| 说明文案                 | 静态                                                            | 与「通知设置」分工：此处管默认，通知页管渠道                 |

从 **通知设置页移除**「新建提醒默认开启通知」行，避免重复。

**（2）日历**

| 项                     | 数据来源                                                      | 行为                                                |
| ---------------------- | ------------------------------------------------------------- | --------------------------------------------------- |
| 新建日程默认带结束时间 | `wx.storage`（`lib/config.ts` 或新建 `lib/content-prefs.ts`） | 开关，默认 `false`（与当前创建页一致）              |
| 默认日程时长           | 本地                                                          | 当开启结束时间时：30 / 60 / 90 分钟 picker，默认 60 |
| 周视图默认选中「今天」 | 本地                                                          | 开关，默认 `true`                                   |

创建页 `event-create` / `event-edit` **读取**上述本地默认值（仅影响新建默认值，不改历史数据）。

**（3）通讯录**

| 项          | 数据来源 | 行为                                                                     |
| ----------- | -------- | ------------------------------------------------------------------------ |
| 列表排序    | 本地     | `按姓名` / `最近更新`（默认姓名）                                        |
| 展示副信息  | 本地     | 开关：列表是否显示公司或 handle（二选一或组合，实现取最小：显示 handle） |
| handle 说明 | 静态     | 短文案 + 链到 `docs/tech-decisions.md` 产品摘要（关于页或页内折叠）      |

联系人列表页 **读取**排序偏好（客户端排序，不新增 API）。

**验收**

- [ ] 我的 →「使用偏好」进入单页，三块均可见可操作
- [ ] 提醒默认通知开关保存后刷新仍正确；通知页无重复项
- [ ] 新建日程/联系人列表行为与偏好一致
- [ ] `pnpm check:wechat` 通过

#### 2. 通知设置页收敛（渠道）

| 保留           | 变更                                                                                                              |
| -------------- | ----------------------------------------------------------------------------------------------------------------- |
| 微信通知总开关 | 保持                                                                                                              |
| 模板未配置提示 | 保持                                                                                                              |
| 免打扰时段     | **0.2.1 若 API 已返回 `quietStart/quietEnd`：增加可编辑时间选择并 PUT**；若实现成本高，则保持只读展示并记入 0.2.2 |
| 新建默认通知   | **移至使用偏好 · 提醒**                                                                                           |

**可选 P0+**：页底增加「最近通知记录」只读列表（`GET /v1/notifications/status?limit=10`），最多 10 条，无记录时隐藏。

#### 3. 账号与安全页 `pages/settings-security/`

替换 placeholder `slot=security`。

| 区块     | 内容                                     |
| -------- | ---------------------------------------- |
| 登录方式 | 微信登录；退出请回「我的」底部           |
| 访问令牌 | 说明 PAT 用途；按钮跳转 `settings-agent` |
| 数据范围 | 不读系统通讯录/日历（与关于页一致）      |
| 隐私     | 链到小程序隐私指引（已有组件能力则复用） |

无新 API。

**验收**

- [ ] 菜单「账号与安全」进入实页，非占位

### P1 — 本批尽量完成

#### 4. 提醒 Tab 信息层次

不改路由，仅 `reminders` 列表 WXML/SCSS：

| 改进         | 说明                                           |
| ------------ | ---------------------------------------------- |
| 分段标题强化 | 待办 / 已完成（若有分段）对比度与间距          |
| 逾期态       | `cell-meta.overdue` 统一走设计令牌             |
| 元信息一行   | 截止时间 + 联系人 handle（有则显示），过长省略 |
| 空态         | 无提醒时简短引导 + 跳转创建                    |

不强制独立「提醒详情页」。

#### 5. 通知设置 · 送达摘要（P1）

- `GET /v1/notifications/status?limit=10`
- 展示：目标标题（或 ID 缩写）、状态、时间
- 失败/无配额状态用 `--todo-danger` / `--todo-text-muted`

## 技术任务分解

### 小程序

1. `lib/content-prefs.ts` — 本地偏好读写（calendar + contacts），key 带 userId 前缀。
2. `pages/settings-content-prefs/*` — 四件套；`app.json` 注册。
3. `lib/settings-menu.ts` — 菜单合并为「使用偏好」单入口；调整「提醒与通知」。
4. `pages/notification-settings/*` — 移除默认提醒行；可选勿扰编辑与 status 列表。
5. `pages/settings-security/*` — 四件套；替换 placeholder 路由。
6. `pages/event-create`、`event-edit` — 读 calendar 本地默认。
7. `pages/contacts/contacts.ts` — 读排序/展示偏好。
8. `lib/api.ts` — 若做 status 列表：封装 `fetchNotificationStatus`。
9. `reminders` — P1 样式与元信息。
10. 删除或降级 `settings-placeholder` 的 `reminders`/`calendar`/`contacts` slot（菜单不再指向）。

### API

- **无必须变更**。可选：文档注明 `defaultReminderEnabled` 推荐在小程序「使用偏好」编辑。
- 免打扰若做编辑：已支持 `PUT` 的 `quietStart`/`quietEnd`，无需新版本号。
