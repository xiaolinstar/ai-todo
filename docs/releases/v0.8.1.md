# v0.8.1 Release Notes

发布目标：设置一致性与账号体验收口 — 微信通知、头像持久性、隐私协议版本。

## 组件版本

| 组件 | 版本 | 说明 |
|------|------|------|
| Git `release_tag` | `v0.8.1` | 体验 patch |
| API | `0.4.1` | 头像空值不覆盖；`clearAvatar` 显式清空 |
| CLI | `0.6.0` | **无变更**（沿用 v0.8.0） |
| 微信小程序 | `0.8.1` | 微信通知设置；隐私版本缓存 |

## 主要变化

- 小程序：**微信通知**合并设置页（单一「微信提醒」开关 + 免打扰时段 + 最近送达子页），同时作用于提醒与日历；从「使用偏好」迁出通知项。
- 小程序：创建提醒/日程不再提供单项微信提醒开关，统一由全局设置控制。
- API：`PATCH /v1/me/profile` 空 `avatarUrl` 不再清空已有头像；`clearAvatar: true` 显式清空。
- 小程序：隐私同意版本本地缓存（`2026-06-07`），同版本重登不重复弹窗。
- 小程序：`miniprogram-ci upload` 暂不启用，发布请用微信开发者工具上传。

## 发布说明

- API 有 patch 变更时需 CD（`release_tag=v0.8.1`）。
- 微信小程序上传 `0.8.1`（开发者工具）。
- **不**发布 CLI npm（本批无 CLI 变更）。

详见 [v0.8.1-plan.md](./v0.8.1-plan.md)。
