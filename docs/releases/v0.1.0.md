# ai-todo v0.1.0 发布说明

发布日期：2026-05-31
发布目标：微信小程序体验版 / 内测版

## 版本概述

`v0.1.0` 是 ai-todo 的第一个可内测运行版本，面向本人和少量朋友测试。它提供轻量的提醒、日历和通讯录能力，支持微信小程序登录使用，也支持通过 CLI / Agent 访问同一套数据。

微信小程序发布版本号填写为 `0.1.0`，Git tag 使用 `v0.1.0`。

## 主要变更

- 新增微信小程序原生 MVP：
  - 提醒列表、创建提醒、完成提醒、左滑删除提醒；
  - 日历今日视图和创建日程；
  - 通讯录列表、创建联系人、联系人选择器；
  - 我的页支持微信登录、连接状态和 CLI / Agent 访问令牌管理。
- 新增生产 API 能力：
  - Reminder / Calendar / Contacts CRUD；
  - 微信登录 identity 绑定；
  - PAT / session token；
  - 用户数据隔离；
  - 幂等写入和 CommandLog 审计。
- 新增 CLI / Agent 能力：
  - `today`、`whoami`；
  - `reminder`、`calendar`、`contact` 结构化子命令；
  - `login` / `logout`；
  - Skill 文档和 `@ai-todo/agent-protocol` 工具表。
- 完成生产部署基础：
  - Docker API 镜像；
  - PostgreSQL + Alembic；
  - GitHub Actions CI / CD；
  - xiaolin-gateway 反代 `https://xingxiaolin.cn`；
  - `/v1/health` 与 `/v1/health/db` 健康检查。
- 提醒触达服务端闭环已准备：
  - 通知偏好、订阅授权结果和投递记录表；
  - 独立 worker 进程；
  - worker 默认放入 `notifications` profile，不随生产 compose 默认启动。

## 内测重点

- 微信登录是否稳定。
- 提醒、日历、通讯录核心流程是否容易理解。
- 左滑删除和二次确认是否自然。
- 朋友是否会使用联系人关联。
- CLI / Agent 访问令牌是否能顺利配置。
- 生产 API 和数据库迁移是否稳定。

## 已知限制

- 真实微信订阅消息投递暂不作为 `0.1.0` 发布门槛；worker 默认不启动，后续单独联调。
- 当前提醒列表以今日视图为主，历史和未来筛选能力有限。
- 删除提醒没有撤销入口。
- 暂无提醒详情页、星标提醒、重复提醒和复杂提醒规则。
- 日程暂无冲突检测和完整参与人管理。
- 内测阶段不提供公开反馈入口或运营后台。

## 微信提审 / 体验版备注

ai-todo 是一款轻量的个人提醒、日历和通讯录小程序。当前版本用于小范围内测，用户可通过微信登录后创建提醒、完成提醒、左滑删除提醒、创建日程和维护个人联系人。小程序请求服务端 API 保存用户数据，服务端已按用户隔离数据，并提供健康检查和生产部署流程。当前版本暂不启用真实微信订阅消息投递，提醒触达能力将在后续版本继续联调。
