# v0.8.10 Release Notes

发布日期：2026-07-12

| 组件              | 版本      | 说明                             |
| ----------------- | --------- | -------------------------------- |
| Git `release_tag` | `v0.8.10` | 当前发布火车                     |

| CLI               | `0.7.1`   | 无变更，沿用 v0.8.9 CLI 能力     |
| 微信小程序        | `0.8.10`  | 未登录错误统一唤起登录           |

## 变更

### 发布可靠性

- K8s app overlay 与 PostgreSQL / PVC 生命周期解耦：应用发布只管理 API、worker、API Service、ConfigMap、Secret。
- PostgreSQL 改由独立 DB overlay 管理，并以 StatefulSet 表达有状态语义。
- Alembic migration 从 API 启动路径中移出，改为通过显式 `db-migration` Job / `DB Migration (K8s)` workflow 执行。
- K8s 清单补齐小型合规项：环境化 `app.kubernetes.io/instance` 标签、worker `startupProbe`、migration Job `resources.requests/limits`。
- K8s CD 增加 API `preStop` 钩子，并修复健康检查中 curl race，降低滚动发布时的短暂失败概率。
- Staging 继续保留 Docker Compose；K8s staging overlay 仅作为未来迁移到更高规格机器时的备用模板。

### 登录体验

- 小程序新增全局未登录错误处理，401 / 未认证错误统一提示并引导登录。
- 联系人、日程、提醒、设置等页面复用统一登录提示，减少各页面分散处理导致的体验不一致。
- 小程序版本号更新至 `0.8.10`。

## 运维说明

- 本批没有新增 Alembic revision；常规发布无需执行 `DB Migration (K8s)`。
- 若后续版本包含 schema 变更，按 [database-migrations.md](../database-migrations.md) 与 [ops-postgresql-data.md](../ops-postgresql-data.md) 先备份，再显式运行 migration workflow。
- `disableNameSuffixHash: true` 暂保留为有意识例外：app、db、migration 三类 overlay 分开执行时需要稳定引用同一组 `ai-todo-config` / `ai-todo-secrets`。
