# v0.5.2 Release Notes

发布目标：触达体验打磨与发布可靠性（通知记录可读、包体合规）。

## 组件版本

| 组件 | 版本 | 说明 |
|------|------|------|
| Git `release_tag` | `v0.5.2` | 本 tag |
| API | `0.2.6` | 通知 status 返回 `targetTitle`；日程 `thing13` 可展示时段 |
| CLI | `0.4.0` | 无变更 |
| 微信小程序 | `0.5.2` | 通知记录标题与状态文案；移除超标 PNG |

## 变更

### API

- `GET /v1/notifications/status` 每条 delivery 增加 `targetTitle`（提醒/日程标题，软删仍保留标题）。
- 日程订阅消息：无地点/描述但有结束时间时，`thing13` 展示 `HH:MM-HH:MM`。

### 微信小程序

- 「通知设置 → 最近送达」显示「提醒/日程 · 标题」及中文状态。
- `no_quota` / `failed` 给出可行动提示。
- 移除未使用的 1024 PNG，满足微信 200KB 代码质量检查。

### 运维

- `apps/api/scripts/dev-up.sh`：本地 Docker + worker 一键启动。
- CI：`pnpm check:wechat` 校验媒体文件 ≤ 200KB。
- `deploy.md` 补充本地启动与生产验收清单。

## 非目标

- 免打扰 worker 实现（仍仅保存 settings，P1 遗留）
- Icon / Tab 品牌 UI（v0.6.0）

## 验证

- `pnpm test:api`、`pnpm check:wechat`
- 提醒 + 日程各创建 1 条订阅消息并联调
- 微信上传代码质量无 200KB 告警
