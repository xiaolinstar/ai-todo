# v0.5.4 发布计划

## 目标

1. **免打扰 worker（方案 A）**：静默时段内不 claim 待发送 delivery，时段结束后正常发送。
2. **提醒列表补全**：小程序展示全部 pending（含未来），按 `due_at` 升序；已完成 Tab 用 `status=completed` + `sort=completed_at`。
3. **发布文档**：本计划、`v0.5.4.md`、兼容矩阵更新。

## 组件版本

| 组件 | 版本 |
|------|------|
| Git `release_tag` | `v0.5.4` |
| API | `0.2.7` |
| CLI | `0.4.2`（无契约变更） |
| 微信小程序 | `0.5.4` |

## API 变更

- `GET /v1/reminders` 新增 `sort`：`created_at`（默认）、`due_at`、`completed_at`。
- `limit` 可选；不传 limit 且 `sort=due_at|completed_at` 时返回全量（CLI 仍显式 `limit=50` + 默认 `created_at` 分页）。
- Worker `claim_due`：读取用户 `quiet_start/quiet_end` + timezone，静默时段 skip claim。

## 小程序

- 提醒页：`fetchReminders({ status: "pending", sort: "due_at" })` + 已完成列表。
- 日期标题仍用 `loadAccountDay()`（`/v1/today` 取账户日历日）。

## 非目标

- 列表分组（逾期/今天/未来）
- 品牌 Tab / Icon（v0.6.0）
- CLI `--all`、新微信模板
