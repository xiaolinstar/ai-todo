# v0.8.3 Release Notes

发布目标：登录保活与体验收口 — session 滑动续期、静默续登、CLI 移除 `token`、微信提醒文案。

## 组件版本

| 组件 | 版本 | 说明 |
|------|------|------|
| Git `release_tag` | `v0.8.3` | 账号体验 patch |
| API | `0.4.2` | session token 滑动续期（`session_token_max_ttl_days` 默认 180） |
| CLI | `0.6.1` | **移除** `ai-todo token`；保留 `login --token` / `login --issue-pat` |
| 微信小程序 | `0.8.3` | `restoreSession`、401 重试、登录态广播 |

## 主要变化

### API

- `resolve_token`：对 `token_type=session` 在认证成功时将 `expires_at` 延长 `session_token_ttl_days`（默认 30 天），上限 `session_token_max_ttl_days`（默认 180 天）。
- PAT 行为不变。

### 微信小程序

- 统一 `restoreSession()`：`app.onLaunch` 与「我的」页共用；token 失效时先静默 `wx.login`，失败再清本地 token。
- `lib/api.ts`：业务请求 401 时静默续登并重试一次；并发 401 合并为单次 `wx.login`。
- `onSessionChange` / `globalData.session`：冷启动续登后「我的」不再短暂闪未登录。
- 微信提醒：关闭开关时说明「已授权待送达」与「不再新授权」的区别。

### CLI

- 删除 `ai-todo token list|create|revoke|revoke-all`；执行 `ai-todo token` 时输出小程序引导。
- PAT 创建/列表/吊销仅在小程序 **我的 → Agent 令牌**。

## 发布说明

- 建议先 CD staging（API `0.4.2` + 体验版 `0.8.3`），真机验收后再晋升 production。
- CLI `0.6.1` 可与同 tag 发布或稍后单独 `npm publish`。

详见 [v0.8.3-plan.md](./v0.8.3-plan.md)。
