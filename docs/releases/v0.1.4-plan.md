# ai-todo v0.1.4 版本规划

日期：2026-06-03  
发布目标：微信小程序体验版 / 内测编辑能力修订版

## 版本定位

`0.1.4` 是 `0.1.3` 发布后的**编辑能力修订版本**。目标是补齐提醒、日历、应用内联系人在小程序端的**信息再编辑（update）**闭环，与已有 FastAPI `PATCH` 接口和 CLI 子命令对齐。

本版本**不扩展数据模型**，不新增隐私类目，优先复用 `0.1.2` / `0.1.3` 已打磨的创建表单样式与交互。服务端以透传现有 update 能力为主；主要工作量在小程序页面、列表入口与 `lib/api.ts` 封装。

微信小程序客户端、FastAPI 服务端和 CLI 继续使用同一产品版本号 `0.1.4`，Git tag 为 `v0.1.4`。

## 现状与差距（0.1.3 基线）

| 能力 | API | CLI | 小程序 |
|------|-----|-----|--------|
| 提醒 update | `PATCH /v1/reminders/{id}` | `reminder update`（仅 title/notes）、`reschedule` | 仅完成、删除 |
| 日历 update | `PATCH /v1/calendar/events/{id}` | `calendar update`（字段较全） | 仅创建 |
| 联系人 update | `PATCH /v1/contacts/{id}` | `contact update`（字段较全） | 仅创建；点击为只读弹窗 |

小程序 `lib/api.ts` 当前缺少：`getReminder`、`updateReminder`、`getCalendarEvent`、`updateCalendarEvent`、`getContact`、`updateContact`（及可选 `rescheduleReminder`）。

`v0.1.3` 发布说明已标明：提醒、日历、通讯录的再编辑规划进入 `0.1.4`。

## 前置条件

开始 `0.1.4` 开发前，应先完成：

- `0.1.3` 已提交体验版或完成一轮内测验收。
- 生产 API `releaseTag=v0.1.3` 可用，smoke 通过。
- 确认内测反馈中「创建后无法改标题/时间/联系人」为高频诉求（与本次范围一致）。

## 版本范围

### P0：0.1.3 发布后回归

- 跑发布后 smoke test（登录、创建提醒/日历/联系人、完成提醒、删除、CLI `today` / `list`）。
- 修复 `0.1.3` 体验版阻塞问题。
- 隐私接口扫描与 `pnpm check:wechat`、`pnpm test:api` 保持绿灯。

验收标准：线上 `0.1.3` 可稳定创建与列表浏览，无登录/同步阻塞。

### P1：小程序 API 封装

在 `apps/miniapp/miniprogram/lib/api.ts` 增加：

- `fetchReminder(id)` → `GET /v1/reminders/{id}`
- `updateReminder(id, input)` → `PATCH /v1/reminders/{id}`
- `rescheduleReminder(id, input)` → `PATCH /v1/reminders/{id}/reschedule`（或与 update 合并调用策略二选一，优先与 CLI 一致保留 reschedule 端点）
- `fetchCalendarEvent(id)` → `GET /v1/calendar/events/{id}`
- `updateCalendarEvent(id, input)` → `PATCH /v1/calendar/events/{id}`
- `fetchContact(id)` → `GET /v1/contacts/{id}`
- `updateContact(id, input)` → `PATCH /v1/contacts/{id}`

类型定义补齐 `notes`（提醒）、`description`（日历）、联系人 `notes` / `methods` 等详情字段，与 API 响应一致。

验收标准：小程序可独立拉取单条详情并 PATCH 成功，错误信息可展示给用户。

### P1：提醒编辑

**入口**

- 提醒列表项点击（非完成圆点、非左滑删除区域）进入「编辑提醒」页。
- 已完成提醒允许编辑标题/备注；改期/联系人策略与产品约定一致（建议：已完成仅允许改文案，改期需先「恢复待办」——本版本可不做恢复，仅隐藏改期控件）。

**页面**

- 新增 `pages/reminder-edit/reminder-edit`（推荐独立页，与 `reminder-create` 表单结构对齐，避免创建页逻辑膨胀）。
- `onLoad` 接收 `id`，拉取详情后预填：标题、备注（折叠）、截止时间、通知开关、关联联系人。
- 保存调用 `updateReminder`；若截止时间或 `remindAt` 变更，调用 `reschedule` 或 update 中带 `dueAt`/`remindAt`（与 API 文档一致，实现时择一）。

**字段范围（与 create 对齐）**

- 标题、备注、`dueAt` / `remindAt`、单个关联联系人（`contactIds` 数组长度为 0 或 1，与创建页一致）。

验收标准：

- 从列表进入编辑 → 修改标题/时间/联系人 → 保存返回列表且数据刷新。
- 长按完成、左滑删除行为不变。
- 10 秒内可完成「只改标题」类小修。

### P1：日历编辑

**入口**

- 日历页日程卡片点击进入「编辑日程」。

**页面**

- 新增 `pages/event-edit/event-edit`，与 `event-create` 字段对齐：标题、开始/结束时间、地点、描述（折叠）、联系人。

**字段范围**

- `title`、`startAt`、`endAt`、`location`、`description`、`contactIds`。

验收标准：

- 修改时间或地点后列表/当日视图展示更新。
- 不引入系统日历读写。

### P1：联系人编辑

**入口**

- 通讯录列表项点击由只读 `showModal` 改为进入「编辑联系人」页（或弹窗内「编辑」按钮 — 优先独立页，与 `contact-create` 一致）。

**页面**

- 新增 `pages/contact-edit/contact-edit`，预填：姓名、邮箱、电话、公司、职位、备注。
- 保存调用 `updateContact`；`methods` 按创建页规则组装（邮箱/电话 primary）。

**字段范围**

- 与 `0.1.3` 创建页相同；不编辑 `handle`（除非产品明确要求，本版本不做 handle 修改 UI）。

验收标准：

- 编辑后返回通讯录列表并刷新。
- 不读取/写入系统通讯录。

### P2：列表与导航体验

- `app.json` 注册三个 edit 页面；导航栏标题：「编辑提醒」「编辑日程」「编辑联系人」。
- 保存成功：`navigateBack` + 上一页 `onShow` 刷新（与 create 页一致）。
- 加载失败、保存失败：toast + 可返回。
- 评估是否在左滑操作中增加「编辑」次要入口（非必须，点击主入口即可）。

验收标准：三 Tab 主流程无死链，返回栈合理。

### P2：CLI 与 Agent 对齐（已完成）

- [x] `reminder update` 补充 `--due`、`--remind`、`--contact`；`readRepeatedFlag` 修复 `--contact` 与 `--json` 相邻解析。
- [x] `docs/agent-usage.md`、`skills/ai-todo/SKILL.md`、`help.ts` 已更新。
- [x] `test_cli_reminder_show_update_delete`、`test_cli_contact_show_update` 集成测试。

验收标准：Agent/CLI 可改时间与联系人，与小程序字段语义一致。

### P2：只读详情页（明确降级）

- **本版本不单独做**纯详情页；编辑页即承担「查看完整信息 + 修改」。
- 若编辑页加载前需要骨架屏，复用 create 页样式即可。

## 建议实施顺序

1. 完成 `0.1.3` 发布与回归。
2. `lib/api.ts` 封装 GET/PATCH（+ reschedule）。
3. **联系人编辑**（表单与 create 最接近，风险低）→ 验收。
4. **提醒编辑**（列表点击入口 + 与完成/左滑手势区分）→ 验收。
5. **日历编辑** → 验收。
6. 隐私扫描、`pnpm check:wechat`、`pnpm test:api`。
7. 撰写 `docs/releases/v0.1.4.md`，打 `v0.1.4` tag，小程序提体验版，服务端 CD。

## 隐私与合规门禁

与 `0.1.3` 相同：不新增隐私类目；不接入系统通讯录/系统日历/手机号/定位/相册等。

发布前执行：

```bash
rg "getPhoneNumber|chooseImage|chooseMedia|getLocation|chooseLocation|getClipboardData|getUserProfile|requestSubscribeMessage|chooseAvatar|type=\"nickname\"|onNeedPrivacyAuthorization|requirePrivacyAuthorize|agreePrivacyAuthorization" apps/miniapp/miniprogram apps/miniapp/types
```

## 明确不做

`0.1.4` 仍不做：

- 批量编辑、多选、拖拽排序。
- 复杂重复提醒/重复日历。
- 撤销删除、已删除列表、星标。
- 系统通讯录/系统日历集成。
- MCP Server、数据模型变更。
- 手机号、定位、上传类能力。

上述能力继续在 `0.2.0` 或后续版本评估。

## 与路线图关系
