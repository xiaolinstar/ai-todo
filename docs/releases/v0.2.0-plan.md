# ai-todo v0.2.0 版本规划

日期：2026-06-03  
发布目标：微信小程序体验版 / 内测体验增强版

## 版本定位

`0.2.0` 在 `0.1.4`（编辑 + 左滑删除）之后，聚焦 **账户中心与内测体验**，不扩展数据模型、不新增隐私类目。

- **P0**：「我的」信息架构、通知设置页、关于/版本、连接状态（开发版）。
- **P1**：提醒/今日呈现与通知策略说明（本批可先做文档与小步 UI，完整详情页可延后）。
- **P2**：星标、撤销删除、独立提醒详情页等（推迟到 `0.2.1` 或 `0.3.0`）。

Git **`release_tag`：`v0.2.0`**（CD 不变）。组件 L1 独立发展，见 [versioning.md](./versioning.md)、[compatibility.md](./compatibility.md)。

## 前置条件

- [x] `v0.1.4` 已发布（tag + 小程序体验版 + 验收）。
- [ ] 内测反馈确认「我的」页信息分散、通知设置难找为高频痛点（可与 P0 并行）。

## 现状（0.1.4 基线）

| 区域 | 已有 | 缺口 |
|------|------|------|
| 我的 · 账户 | 资料卡、登录/退出、个人资料入口、首次资料确认 | 无统一「设置」入口；资料卡暴露原始 userId |
| 我的 · Agent | PAT 创建/列表/吊销（含确认） | 分组与文案偏长；与账户设置混在一起 |
| 通知 | API `GET/PUT /v1/notifications/settings`；创建提醒页可订阅 | **无**独立通知设置页；编辑提醒不重新订阅（已知限制） |
| 关于 | 无 | 无小程序版本、API 版本、帮助说明 |
| 连接状态 | 登录失败 toast | 开发版无 API 地址/健康状态展示 |

## 版本范围

### P0：我的 / 设置中心（本批实现，迭代中）

**信息架构（设置枢纽）**

「我的」仅保留：资料头图、分组菜单（二级页）、底部退出登录。配置由 `lib/settings-menu.ts` 驱动。

| 分组 | 入口 | 状态 |
|------|------|------|
| 账户 | 个人资料、时区、账号与安全 | 资料已实现；后两者占位 |
| 提醒与通知 | 通知设置、提醒偏好 | 通知已实现；偏好占位 |
| 日历与通讯录 | 日历偏好、通讯录偏好 | 占位 |
| 扩展 | CLI / Agent 接入 | 已实现（`settings-agent`） |
| 关于 | 关于 ai-todo | 已实现 |
| 开发 | 开发者选项 | 仅 develop（`settings-dev`） |

**已实现二级页**

- `notification-settings` — `PUT /v1/notifications/settings`
- `settings-agent` — PAT 管理（从「我的」迁出）
- `about` — 版本、health、用户 ID
- `settings-dev` — API 地址/连通性、开发工具
- `settings-placeholder?slot=` — 统一占位壳

**后续小版本**

- **`v0.2.1`**：提醒/日历/通讯录合并为「使用偏好」单页；通知设置仅渠道；账号与安全实装。详见 [v0.2.1-plan.md](./v0.2.1-plan.md)。
- **`v0.2.2`**：时区与 `username` 的 API `PATCH` + 按用户名建联联系人。

验收标准：

- [x] 「我的」仅为菜单 + 退出，无 PAT/长文案堆叠
- [ ] 各入口跳转正确（已实现 vs 占位）
- [x] `pnpm check:wechat` 通过

### P1：提醒体验（可选，本批或 0.2.1）

- 提醒列表/今日信息层次优化（不强制独立详情页）。
- 通知策略文档：编辑提醒与订阅关系写在 `v0.2.0.md` 已知限制。
- 评估 `GET /v1/notifications/status` 是否在「我的」展示配额摘要（只读）。

### P2：明确推迟

- 提醒星标、撤销删除、已删除列表。
- 复杂重复提醒/日历规则。
- MCP Server、系统通讯录/日历。

## 建议实施顺序

1. 撰写 `v0.2.0-plan.md`、`v0.2.0.md`（本文件与发布说明）。
2. `lib/version.ts` + `api` 封装；注册新页面。
3. 实现 `notification-settings`、`about`；重构 `mine` 入口。
4. 开发版连接状态；`pnpm check:wechat`。
5. 内测验收 → bump miniapp `0.2.0` → tag `v0.2.0`（api 无变更可保持 `0.1.3`）。

## 隐私与合规门禁

与 `0.1.4` 相同：不新增隐私类目。通知设置仅读写已有服务端字段，不新增 `requestSubscribeMessage` 调用点（仍仅在创建提醒流程）。
