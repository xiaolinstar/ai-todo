## 问题陈述

当前提醒仅有 `pending` / `completed` / `cancelled`（内部）。用户反馈部分事项需要**处理中**阶段：已开始但未完成，与「尚未开始」区分。该变更贯穿 DB、API、小程序、CLI、Agent/MCP，不宜与 v0.7.0 体验修复混在同一 tag。

## P0：状态模型与迁移

### API（`0.4.0`）

- 合法 `status`：`pending`、`in_progress`、`completed`、`cancelled`。
- 创建默认仍为 `pending`。
- `PUT /v1/reminders/{id}` 与 by-source update 支持 `status: in_progress`。
- `GET /v1/reminders?status=` 接受 `in_progress`；`pending` 行为不变。
- `completed_at`：进入 `completed` 写入；离开 `completed` 清空。
- Alembic 迁移：无需改列类型（仍为 string）；可选数据修复脚本（无历史 `in_progress`）。
- OpenAPI / `docs/api-design.md` 更新。

### 共享类型（`@ai-todo/shared`）

```ts
export type ReminderStatus = "pending" | "in_progress" | "completed" | "cancelled";
```

### 小程序（`0.8.0`）

- 提醒 Tab：段筛选 **未完成**（`pending`）/ **处理中**（`in_progress`）/ **已完成**（`completed`）。
- 列表行内或详情：一键「标为处理中」「标为完成」；与左滑/现有完成不冲突。
- 逾期/今天/未来分组：未完成类 = `pending` + `in_progress`。
- 微信提审备注说明「待办状态细分，无新隐私类目」。

### CLI（`0.6.0`）

- `reminder list --status pending|in_progress|completed|cancelled`
- `reminder update <id> --status in_progress`（及 by-source 变体）
- 人类可读输出英文状态标签：`pending` → `Not started`，`in_progress` → `In progress`，等。
- `--json` 仍输出 API 原始 `status` 字符串。

### Agent 面

- `agent-protocol`：tools + guidelines 增补 `in_progress`。
- `skills/ai-todo/SKILL.md`：工单进行中 → `in_progress` 示例。
- MCP：`reminder_list` / `reminder_update_by_source` 的 status schema 扩展；npm `@xiaolinstar/ai-todo-mcp@0.2.0`。
