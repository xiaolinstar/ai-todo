## 变更

### 发布可靠性

- 修复 K8s CD rollout 后立即读取旧实例 `/v1/health` 导致 `gitSha mismatch` 的 race：部署脚本现在会在健康检查循环内校验 `gitSha`，直到命中新版本或超时。
- 将 K8s 发布身份写入独立 `.env.production.release`，与人工维护的环境配置分离，避免 release metadata 覆盖长期配置。
- 将 K8s overlay 环境文件从 `env-configs` 统一为 `env-config`，降低命名歧义。
- API deployment 增加 `preStop` 等待，减少滚动发布期间旧实例与 NodePort / Service 切换造成的短暂不一致。
- 回退 CD / CD (K8s) 为纯手动 `workflow_dispatch`，明确 `release_tag` 只是运行参数，不再通过 tag push 自动触发部署。
- 删除半成品 `release-please` workflow，避免与当前手工发布火车、staging 晋升和小程序审核节奏冲突。

### 小程序体验验收

- 小程序可见版本号从 `0.8.10` 同步到 `0.8.13`，覆盖 `package.json`、`project.config.json` 与「关于」页读取的 `MINIAPP_VERSION`。
- 更新小程序 Agent 引导中的当前版本，避免后续维护继续按旧版本做上传/验收说明。

## 运维说明

- 本批没有新增 Alembic revision；常规发布无需执行 `DB Migration (K8s)`。
- `release_tag=v0.8.13` 已通过 `CD (K8s)` 手动发布到 production-k8s。
- 后续日常发布继续遵循：CI 绿并产出 `deploy-manifest` 后，人工 Run workflow 并填写 `release_tag`；`ci_run_id` 仅用于回滚或救急。
