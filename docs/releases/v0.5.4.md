# v0.5.4 Release Notes

发布目标：免打扰 worker 生效 + 提醒列表全量 pending 按时间排序。

## 组件版本

| 组件 | 版本 | 说明 |
|------|------|------|
| Git `release_tag` | `v0.5.4` | 本 tag |
| API | `0.2.7` | 提醒 list `sort`；worker 静默时段 skip claim |
| CLI | `0.4.2` | 无变更 |
| 微信小程序 | `0.5.4` | 全部 pending + 已完成历史，按时间排序 |

## 变更

### API

- `GET /v1/reminders` 支持 `sort=created_at|due_at|completed_at`；`due_at` / `completed_at` 可不传 `limit` 全量返回。
- 通知 worker：`claim_due` 在用户免打扰时段内跳过 claim，delivery 保持 `pending` 直至时段结束。

### 微信小程序

- 提醒页待办：全部 pending（含未来与逾期），按截止时间升序，无 due 排最后。
- 已完成 Tab：`status=completed` + `completed_at` 降序。
- 页头日期仍按账户时区「今天」展示。

## 非目标

- 列表分组 UI
- Icon / Tab 品牌（v0.6.0）

## 验证

- `pnpm test:api`、`pnpm check:wechat`
- 免打扰时段内创建到期提醒，确认不发送、时段外正常 claim
- 小程序待办含未来日期项且排序正确
