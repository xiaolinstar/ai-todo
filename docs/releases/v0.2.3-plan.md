**主题：** 生产 CD 从 2 步 Job 扩展为 **6 步逻辑**，以公网拨测/黑盒为发布门禁，失败自动回滚并出报告。

## P0 交付项 — CD 六步（已实现）

| 步骤 | Job / 脚本 |
|------|------------|
| 1 解析 manifest | `resolve-manifest` |
| 2 部署 | `deploy` → `deploy-from-manifest.sh` |
| 3 发布后测试 | `post-deploy-verify` → `scripts/cd/post-deploy-verify.py` |
| 4 回滚 | `rollback` → `AI_TODO_CD_ACTION=rollback` / `--rollback-to-previous` |
| 5 回滚后验证 | `post-rollback-verify` |
| 6 发布报告 | `release-report` → `scripts/cd/write-release-report.mjs` |

详见 **[v0.2.3-cd-pipeline-plan.md](./v0.2.3-cd-pipeline-plan.md)**、[ci-cd.md](../ci-cd.md)。

| 分期 | 内容 |
|------|------|

| P1 | L2 黑盒（`CD_SMOKE_PAT`） |
| P2 | L3 E2E marker / 全量生产只读回归 |

## 发布流程（有 CD 变更时）

1. 合并 CD + 部署脚本到 `main`，staging 手动 CD 验收。
2. 若无客户端变更：可 **仅打 `v0.2.3` tag 用于叙事**，miniapp 不必上传。
3. production CD：`release_tag=v0.2.3`，确认 ⑥ 报告为 success。
4. 更新 `compatibility.md`（若仅运维/CD，说明「无 API 契约变更」）。
