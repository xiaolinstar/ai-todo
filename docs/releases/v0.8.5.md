# v0.8.5 Release Notes

发布目标：低频使用体验补丁 — 小程序详情页长标题完整可读，并保留 CLI/API 写入的多联系人关联。

| ----------------- | -------- | ----------------------- |
| Git `release_tag` | `v0.8.5` | 体验 patch              |

| 微信小程序        | `0.8.5`  | 提醒/日程详情页体验修复 |

## 主要变化

### 微信小程序

- 提醒详情页和日程详情页的标题从单行输入改为多行输入，长标题可完整阅读。
- 标题输入框使用受控高度，限制在 2 到 5 行，避免 `textarea auto-height` 偶发撑得过大。
- 提醒/日程编辑页保留完整 `contacts[]`，不再只取第一个联系人。
- 多联系人显示为 `张三、李四等 2 人` 这类汇总文案。
- 未改联系人直接保存时，`contactIds` 会提交完整联系人列表，避免 CLI/API 创建的多联系人事项被小程序保存后丢失。
- 若用户在当前单选联系人 picker 中选择新联系人，仍会用新选中的单个联系人替换原联系人列表；多选 picker 留作后续增强。
