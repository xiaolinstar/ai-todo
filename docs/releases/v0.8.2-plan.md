## 问题陈述

内测反馈与代码审查显示，**v0.8.1 未覆盖「会话本身」**：

| 现象 | 根因（当前实现） |
|------|------------------|
| 隔数周再打开需重新点「微信登录」 | 微信 session token **固定 30 天**过期（`session_token_ttl_days=30`），且**不随使用续期** |
| 本地 token 失效后「我的」立刻显示未登录 | `mine.refreshSession` 在 `fetchMe` 失败时 **立即 `clearToken`**，未先尝试静默 `wx.login` |
| 启动时可能已静默登入，但「我的」仍显示未登录 | `app.onLaunch` 的 `ensureAuth()` 与 `mine.onShow` 的 `refreshSession` **竞态**，无统一会话状态广播 |
| 用户以为关掉「微信提醒」就不再推送 | Worker 不检查 `wechatEnabled`；仅影响新建/编辑时的订阅授权（**预期需在设置文案或行为上对齐**） |

v0.8.1 已解决的是**重登副作用**（头像被覆盖、隐私重复弹窗），不是**长期免登录**。

## 产品决策（实施前建议默认）

| # | 问题 | 建议（默认） |
|---|------|----------------|
| 1 | 会话续期策略？ | **滑动续期**：每次带 session token 的成功 API 请求将 `expires_at` 延长 30 天（上限可选 180 天）；或简单将默认 TTL 调至 **90 天** 且每次 `resolve_token` 续期。 |
| 2 | token 过期后小程序行为？ | **先静默 `wx.login` 换新 token**，失败再展示未登录；不在首轮 401 就清本地 token。 |
| 3 | 「我的」与全局会话如何同步？ | `ensureAuth` / 静默续登成功后通过 **轻量事件或 `getApp` 回调** 通知各 Tab 刷新登录态。 |
| 4 | 关闭「微信提醒」是否取消已在途推送？ | **本批仅改文案**说明「关闭后不再为新事项请求授权；已授权项仍可能送达」；取消 pending delivery 作为 **P2 / v0.9 前** 可选。 |
| 5 | CLI `token list` 是否捆绑本 tag？ | **可拆**：小程序 + API 先发布 `v0.8.2`；CLI `0.6.1` 可同 tag 或稍后单独 `npm publish`。 |

## P0 — 登录保活（小程序 + 可选 API）

### 小程序 — 环境隔离（v0.8.2 已落地）

体验版 / 正式版 / 开发版交替使用时，本地 `token`、`privacyConsentVersion`、`profileSetupSeen` **按 API 环境分桶**（`lib/config.ts` → `getStorageScope`），避免 staging token 误用于 production。

- `syncRuntimeConfig()`：`onLaunch` 与 `mine.onShow` 对齐 `apiUrl` 并迁移旧版扁平键
- 关于页展示运行环境标签与 API 地址，便于内测确认当前连的是哪套后端
- 各环境隐私同意独立记录：首次登录 staging 仍会走隐私授权（若该环境尚未同意）
- **develop 三分流**：模拟器（`platform=devtools`）→ 本地 `127.0.0.1:3100`；手机预览/真机调试 → staging（与体验版同后端）

发布流程（staging → production、提审并行）见 [staging-production-promotion.md](./staging-production-promotion.md)。

### API（若采用滑动续期）

- `resolve_token`：对 `token_type=session` 在认证成功时更新 `expires_at`（如 `now + session_token_ttl_days`）。
- 可选：配置项 `session_token_max_ttl_days` 防止无限续期。
- 测试：过期边界、续期后仍有效、PAT 行为不变。

### 小程序

- **统一会话恢复路径**：抽取 `restoreSession()`（有 token → `fetchMe`；失败 → 静默 `wx.login` → 保存新 token → `fetchMe`）。
- `app.onLaunch`：`ensureAuth` 改为调用 `restoreSession`，失败不抛未处理异常。
- `mine.onShow`：使用同一 `restoreSession`，避免与 launch 竞态导致误清 token。
- **401 处理**：全局 `request` 层对 session 过期尝试一次静默续登后重试（需防循环）；仍失败再清 token 并引导登录。
- **UI**：未登录时「我的」副标题与受保护设置页入口保持一致；续登成功无多余 toast（仅用户主动点登录时 toast）。

### 验收

- [ ] 体验版 ↔ 正式版切换后，「我的」无需下拉刷新即可显示对应环境登录态（各环境 token 独立）
- [ ] 关于页可看到运行环境与 API 地址
- [ ] staging 首次登录会弹出隐私授权（与 production 同意记录互不影响）
- [ ] 30 天内未打开：重新进入小程序，**无需手点「微信登录」**即可在「我的」看到已登录态（在 token 未过期或续期策略生效前提下）。
- [ ] token 已过期但微信侧仍可 `wx.login`：静默续登成功，数据可访问。
- [ ] token 彻底失效：展示「微信登录以使用全部设置」，无原始 `Authorization` 文案。
- [ ] PAT / 开发者工具 dev auth 行为不退化。
