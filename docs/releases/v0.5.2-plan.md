## 主题：「能稳定触达 + 看得懂 + 发得出去」

v0.5.0～0.5.1 完成了 **能力从 0 到 1**；v0.5.2 聚焦 **内测可运维、用户可理解**。

```text
用户创建/改期 → 订阅授权 → delivery pending
        ↓
worker（尊重免打扰）→ 微信送达 → 深链编辑
        ↓
「通知设置」里看得懂发了什么、成功还是失败
```

## 产品决策（待确认）

| # | 问题 | 选项 | 建议 |
|---|------|------|------|
| 1 | 免打扰实现方式 | A. 静默时段内不 claim，结束后立即发；B. 推到静默结束时刻再发 | **A**（实现简单，不篡改用户设定的 start_at/due_at 语义） |
| 2 | 通知记录标题来源 | API join vs 小程序二次请求 | **API join**（一次加载、离线友好） |
| 3 | v0.5.2 是否含免打扰 | 纳入 P0 vs 仅 P1 | 若内测无人设置免打扰 → **P1**；若已有用户设置 → **P0** |
| 4 | 日程消息「截止日期」标签歧义 | 继续容忍 vs 申请新模板 | **继续容忍**；收集 2 周内反馈再议 v0.6 |

确认 #3 后即可按 P0 开工。


## 验收标准

- [ ] 微信上传代码质量：无单文件 > 200KB
- [ ] 「通知设置 → 最近记录」可见提醒/日程 **标题**
- [ ] 提醒、日程订阅消息生产/体验版各测通 1 条
- [ ] worker 容器 `healthy`；`pnpm test:api`、`pnpm check:wechat` 通过
- [ ] （若做免打扰）静默时段内不打扰，结束后仍能收到
