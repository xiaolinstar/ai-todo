# ai-todo v0.1.1 发布说明

发布日期：2026-06-01
发布目标：微信小程序体验版 / 内测修订版

## 版本概述

`v0.1.1` 是 `v0.1.0` 后的内测稳定修订版。这个版本重点修复真实微信登录、头像昵称资料、CLI 资料更新、联系人删除闭环和生产发布节奏问题，让小程序客户端与 FastAPI 服务端保持一致发布。

微信小程序发布版本号填写为 `0.1.1`，Git tag 使用 `v0.1.1`。

## 主要变更

- 优化微信小程序登录后的资料流程：
  - 支持首次登录后确认头像和昵称；
  - 顶部个人信息展示头像昵称；
  - 登录后不再重复弹出资料确认；
  - 开发环境提供“重新设置头像昵称”入口。
- 新增服务端当前用户资料接口：
  - `PATCH /v1/me/profile`；
  - `GET /v1/me` 返回 `avatarUrl`；
  - 新增 `users.avatar_url` migration。
- 新增 CLI 个人资料能力：
  - `ai-todo profile update --name <text> [--avatar-url <url>]`。
- 补齐联系人删除闭环：
  - `DELETE /v1/contacts/{contact_id}`；
  - `ai-todo contact delete <contact_id_or_handle>`；
  - 联系人搜索支持 handle 归一化，`codex-cli` 可命中 `codexclitest...`。
- 调整发布策略：
  - CI 继续在 `main` push / PR 自动运行；
  - 生产 CD 改为仅手动触发；
  - 手动 CD 使用发布 tag 作为入口，默认解析 tag 指向 commit 的成功 CI，回滚时才需要指定 `ci_run_id`；
  - 避免服务端自动部署与小程序人工发布节奏不一致。

## 已知限制

- 微信头像当前仍可能是微信临时地址，后续需要评估上传到服务端或对象存储。
- 个人资料修改页尚未正式产品化，目前主要依赖首次登录资料确认与 CLI。
- 真实微信订阅消息投递仍未作为本版本发布门槛。
- 生产部署仍是单实例替换，后续需要规划蓝绿/灰度发布以减少停机风险。

## 微信提审 / 体验版备注

ai-todo 是一款轻量的个人提醒、日历和通讯录小程序。当前版本用于小范围内测，用户可通过微信登录后同步个人提醒、日程和联系人数据；首次登录会确认微信头像和昵称作为小程序账户展示信息。服务端支持生产 PAT，用户可通过 CLI / Agent 访问同一套数据。本版本补齐了联系人删除能力，并将生产 CD 改为手动触发，以保证小程序客户端和服务端发布节奏一致。
