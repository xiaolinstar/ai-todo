# v0.5.1 Release Notes

发布目标：日程（Calendar Event）接入微信订阅消息，与 v0.5.0 提醒触达共用模板 #15788。

## 组件版本

| 组件 | 版本 | 说明 |
|------|------|------|
| Git `release_tag` | `v0.5.1` | 本 tag |
| API | `0.2.5` | 日程 delivery / worker / sync |
| CLI | `0.4.0` | 无变更 |
| 微信小程序 | `0.5.1` | 日程订阅、改期重新授权、深链、通知记录文案 |

## 变更

### API / Worker

- 新增 `calendar_event` 目标类型与 `calendar_event_start` 模板键。
- `ensure_delivery_for_target` / `sync_calendar_event_target`：日程 `scheduled_at = start_at`。
- 日程 update / delete 后同步 notification delivery 队列。
- Worker 构建日程消息：`thing23` 标题、`time2` 开始时间、`thing13` 地点/描述/默认文案。
- 点击落地页：`pages/calendar/calendar?eventId=`。

### 微信小程序

- 创建日程：订阅授权 + 写入 delivery。
- 编辑日程改开始时间：重新 `requestSubscribeMessage`。
- 订阅消息点击：日历 Tab 深链至 `event-edit`。
- 「使用偏好」：默认微信通知文案覆盖提醒与日程。
- 「通知设置 → 最近记录」：`calendar_event` 显示为「日程」。
