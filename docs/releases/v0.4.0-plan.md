# ai-todo v0.4.0 版本规划 — npm 全局安装与配置优先 UX

## 目标

1. **`@ai-todo/cli` 发布到 npm**，支持 `npm install -g @ai-todo/cli`。
2. **单包打包**：esbuild 将 `@ai-todo/api-client`、`@ai-todo/shared` 打入 CLI 产物，不单独发布 workspace 包。
3. **配置优先**：用户编辑 `~/.ai-todo/settings.json` 即可使用；`login` / `token` / `logout` 从公开 help 移除（实现保留兼容）。
4. **小程序 onboarding**：创建令牌后复制 JSON 配置片段，而非 `ai-todo login` 命令。

## 组件版本

| 组件 | L1 版本 | 说明 |
|------|---------|------|
| CLI | `0.4.0` | npm 发布、esbuild bundle、help/文档 |
| 微信小程序 | `0.4.0` | 复制 settings JSON |
| API | 不变（`0.2.0`） | 无契约变更 |

Git `release_tag` 建议：`v0.4.0`

## npm 发布

```bash
cd apps/cli
pnpm build
npm publish --access public
```

前置：npm 账号已登录；包名 `@ai-todo/cli` 在 registry 可用。

发布物：

- `dist/index.js`（bundle + sourcemap）
- `settings.example.json`
- `README.md`

## 用户路径

```
npm install -g @ai-todo/cli
小程序创建 PAT → 复制 JSON → ~/.ai-todo/settings.json
ai-todo today
```
