# ai-todo v0.6.2 版本规划 — MCP Server（Agent 零 shell）

日期：2026-06-08  
发布目标：在 **v0.6.1**（工具目录 + Skill + 文档）之后，交付可运行的 **stdio MCP Server**，让 Cursor / Claude Desktop 等宿主**无需手写 shell** 即可调用 ai-todo 结构化能力。

## 版本定位

| 组件                             | 目标 L1  | Git tag  | 是否变更                           |
| -------------------------------- | -------- | -------- | ---------------------------------- |
| Git `release_tag`                | `v0.6.2` | 同上     | 新 MCP 包                          |
| API                              | `0.3.0`  | 同上     | **否**                             |
| CLI                              | `0.5.0`  | 同上     | **否**（MCP 子进程调用现有 CLI）   |
| 微信小程序                       | `0.6.0`  | 同上     | **否**                             |
| `@ai-todo/agent-protocol`        | `0.4.0`  | 同上     | **否**（或 patch 仅加 MCP 元数据） |
| `@xiaolinstar/ai-todo-mcp`（新） | `0.1.0`  | npm + 仓 | **是**                             |

前置：**v0.6.1** 已发布；`agent-tools.json` 与 CLI 来源能力可用。

### 与 v0.7.0 的关系

| 火车                         | 主题                      | 面向           |
| ---------------------------- | ------------------------- | -------------- |
| **v0.6.2（本批，推荐先做）** | MCP Server                | Agent / 开发者 |
| **v0.7.0（候选下一批）**     | 小程序来源角标 + 列表分组 | 微信内测用户   |

两批可并行筹备，但 **默认先合 v0.6.2**，避免 Agent 面与小程序 UI 混在同一 tag。

---

## 问题陈述

v0.6.1 交付了 `AI_TODO_AGENT_TOOLS` 与 Skill，但宿主仍须：

```bash
ai-todo reminder find --source email --external-id "..." --json
```

部分环境（Cursor MCP、Claude Desktop 工具面板）更适合 **原生 MCP tools**，而不是让模型拼 shell。仓库尚无 `apps/mcp`，路线图「Skills / MCP」仍缺可执行产物。

## P0：最小 stdio MCP Server

### 架构

```text
Cursor / Claude Desktop
    → MCP stdio (@ai-todo/mcp)
        → spawn: ai-todo <args> --json
            → HTTPS xingxiaolin.cn (PAT from env / settings.json)
```

- **不**在 MCP 内嵌 HTTP 客户端（第一版）；复用 CLI 鉴权与错误码，减少双实现。
- 工具列表从 `packages/agent-protocol/dist/agent-tools.json` 读取（构建时复制或运行时 resolve）。
- 每个 MCP tool 映射一条 CLI 命令模板；参数由 JSON schema 描述。

### 包结构（建议）

```text
packages/mcp/          # 或 apps/mcp/
  package.json         # @ai-todo/mcp, bin: ai-todo-mcp
  src/server.ts        # @modelcontextprotocol/sdk Server + StdioServerTransport
  src/cli-bridge.ts    # 安全 spawn ai-todo，解析 stdout JSON
  src/tool-registry.ts # 从 agent-tools.json 生成 MCP tool definitions
```

### 首批暴露工具（与 v0.6.1 对齐，可全量或子集）

**P0 子集（建议先 12 个，覆盖 80% Agent 场景）**：

| MCP tool                      | CLI                                          |
| ----------------------------- | -------------------------------------------- |
| `whoami`                      | `whoami`                                     |
| `today`                       | `today`                                      |
| `reminder_find`               | `reminder find --source … --external-id …`   |
| `reminder_create`             | `reminder create …`                          |
| `reminder_create_sourced`     | `reminder create --source … --external-id …` |
| `reminder_list`               | `reminder list`                              |
| `reminder_list_by_source`     | `reminder list --source …`                   |
| `reminder_update_by_source`   | `reminder update --source …`                 |
| `reminder_complete_by_source` | `reminder done --source …`                   |
| `contact_search`              | `contact search`                             |
| `calendar_today`              | `calendar today`                             |
| `calendar_create`             | `calendar add …`                             |

其余工具 v0.6.2.1 或配置开关 `AI_TODO_MCP_FULL_TOOLS=1` 再开。

### 认证

| 方式                                | 说明                          |
| ----------------------------------- | ----------------------------- |
| `AI_TODO_TOKEN` + `AI_TODO_API_URL` | Agent / CI 推荐               |
| `~/.ai-todo/settings.json`          | 与 CLI 一致，继承给子进程环境 |

MCP 启动时检查 token；缺失则返回明确错误，不静默失败。

- `docs/mcp-setup.md`（新）：Cursor `mcp.json`、Claude Desktop 配置示例。

- `skills/ai-todo/SKILL.md` 注明：有 MCP 时优先 MCP tools，无则 shell。
