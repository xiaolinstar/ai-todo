# ai-todo 0.1.0 内测上线规划

日期：2026-05-31

## 版本定位

`0.1.0` 是 ai-todo 第一个可内测运行版本，目标是在正式申请的小程序 AppID、生产 API 和真实微信登录环境下，让本人和少量朋友可以稳定使用核心流程。

本版本不追求公开发布的完整运营能力，优先验证：

- 微信小程序能稳定登录并访问生产 API；
- 提醒、日历、通讯录三个核心对象可创建、查询和删除；
- CLI / Agent 能通过 PAT 管理同一套数据；
- 生产部署、迁移、健康检查、回滚文档可执行；
- 提醒触达服务端闭环已具备，但真实微信订阅消息投递可后置联调。

微信小程序发布版本号填写为 `0.1.0`，Git tag 建议使用 `v0.1.0`。

## 核心范围

### 微信小程序

- 微信登录与本地 token 保存。
- 提醒列表、创建提醒、完成提醒、左滑删除提醒。
- 日历今日视图、创建日程。
- 通讯录列表、创建联系人、联系人选择器。
- 我的页：连接状态、微信登录、CLI / Agent 访问令牌管理。
- 本地 AppID 配置采用 `project.private.config.json`，共享 `project.config.json` 保持 `touristid`。

### API / 数据

- FastAPI + PostgreSQL + Alembic 生产部署。
- Reminder / Calendar / Contacts CRUD。
- 微信登录 identity 绑定。
- PAT / session token。
- 用户数据隔离。
- 幂等写入与 CommandLog 审计。
- 通知偏好、订阅授权结果和投递记录表已准备。

### CLI / Agent

- `today`
- `whoami`
- `reminder create/list/done/reschedule/delete`
- `calendar add/today/show/update/delete`
- `contact list/search/show/add/update`
- `login/logout`
- Skill 文档与 `@ai-todo/agent-protocol` 工具表。

### 部署 / 运维

- API Docker 镜像。
- GitHub Actions CI / CD。
- xiaolin-gateway 反代 `https://xingxiaolin.cn`。
- `/v1/health` 与 `/v1/health/db` 健康检查。
- 生产 `.env.production` 模板。
- 数据库迁移安全检查。

## 非目标

- 不在 `0.1.0` 要求真实微信订阅消息完整投递上线；worker 已放入 `notifications` profile，默认不启动。
- 不做提醒详情页。
- 不做星标提醒。
- 不做撤销删除或最近删除列表。
- 不做重复提醒、复杂提醒规则、日程冲突检测。
- 不做 MCP Server 独立包。
- 不做公开运营后台、用户反馈系统或多环境灰度平台。

## 上线前必须完成

### 微信公众平台

- [ ] 使用正式小程序 AppID 导入 `apps/miniapp`。
- [ ] 本地 `project.private.config.json` 配置真实 AppID。
- [ ] 公众平台 request 合法域名配置 `xingxiaolin.cn`。
- [ ] AppSecret 填入生产 `.env.production`。
- [ ] 体验成员加入微信小程序后台。
- [ ] 开发者工具上传体验版。

### 生产 API

- [ ] `AI_TODO_ALLOW_DEV_AUTH=false`。
- [ ] `AI_TODO_WECHAT_APP_ID` 与小程序 AppID 一致。
- [ ] `AI_TODO_WECHAT_APP_SECRET` 正确。
- [ ] `POSTGRES_PASSWORD` 已替换为强密码。
- [ ] `curl https://xingxiaolin.cn/v1/health` 返回 `ok`。
- [ ] `curl https://xingxiaolin.cn/v1/health/db` 返回 `status=ok`，迁移版本为最新。

### 本地与 CI 验证

发布候选前至少运行：

```bash
pnpm typecheck
pnpm build
pnpm check:wechat
pnpm test:api
```

如仅小程序文案或样式微调，至少运行：

```bash
pnpm check:wechat
```

## 体验版验收清单

测试成员至少覆盖以下流程：

- [ ] 首次打开小程序，微信登录成功。
- [ ] 创建一条今天的提醒。
- [ ] 完成提醒，提醒移动到已完成。
- [ ] 左滑删除提醒，二次确认后从列表消失。
- [ ] 创建一条日程，日历页可见。
- [ ] 创建一个联系人。
- [ ] 创建提醒时关联联系人。
- [ ] 我的页显示当前连接状态正常。
- [ ] 生产 API 健康检查正常。
- [ ] CLI 使用 PAT 能 `whoami` 和 `today --json`。

## 已知限制

- 提醒触达 worker 默认不启动；即使创建了订阅授权记录，也不会真实投递微信服务通知。
- 微信订阅消息模板 ID 尚需在公众平台申请并与 worker 字段映射联调。
- 删除提醒目前没有撤销入口。
- 提醒列表只聚焦今日视图，历史和未来提醒筛选能力有限。
- 日程还没有详情页、冲突检测和参与人完整管理。
- 内测阶段不提供公开客服、反馈入口或隐私政策自动化页面。

## 发布说明草案

`0.1.0` 是 ai-todo 的第一个内测版本。它提供轻量的提醒、日历和通讯录能力，支持微信小程序登录使用，也支持通过 CLI / Agent 访问同一套数据。内测阶段主要面向本人和朋友验证真实使用流程、登录稳定性、生产部署和数据隔离。当前版本暂不启用真实微信订阅消息投递，提醒触达能力将在后续版本继续联调。

## 后续观察重点

- 微信登录是否稳定，是否出现 identity / token 异常。
- 朋友是否理解“提醒”和“日历”的区别。
- 今日提醒列表是否足够，是否需要未来/全部筛选。
- 左滑删除是否容易误触，二次确认是否打扰。
- 联系人关联是否真的被使用。
- CLI / Agent 是否需要更明显的 onboarding。
- 订阅消息是否值得作为 `0.2.0` 的核心目标。
