# ai-todo 微信小程序隐私报备清单

本文用于微信公众平台「小程序设置 → 基本设置 → 服务内容声明 → 用户隐私保护指引」配置。重点是对齐后台弹窗里的「使用用户信息类型」，避免把本应用内部数据误报成微信/手机系统权限。

本文不是法律意见。后台选项名称可能随微信公众平台调整，最终以微信公众平台当前页面为准。

## 填写须知

1. **标准句式**：「使用说明」须写为 **开发者收集你的……，用途是……**。授权目的与使用场景均写在「用途是」之后（例如「……，在你……时使用」），勿另起「收集场景为……」分句。
2. **句末句号**：微信后台会在粘贴内容后自动拼接句号（`。`）。本文可复制文案**句末勿加句号**，避免出现两个句号。
3. **自定义条目**：后台「隐私接口」选「自定义」时，按下方表格的「信息名称 + 使用说明」逐条新增。
4. **生效验证**：保存后在体验版真机点击「查看隐私保护指引」，确认内容非空后再提审。
5. **与代码一致**：发布前运行文末 `rg` 检查，新增隐私 API 须先更新本文再改后台。

## 当前结论

`ai-todo` 当前是个人提醒、日历和联系人管理工具。小程序实际使用的微信隐私相关能力较少：

- 使用微信登录凭证完成账号登录。
- 用户可主动选择微信头像和昵称作为小程序账户资料。
- 用户可创建提醒、日历和本应用内联系人。
- 用户可在输入框中自行粘贴文本，用于创建提醒或日历。
- 用户可创建 CLI 访问令牌，并主动复制到剪贴板。
- 如配置订阅消息模板，创建提醒或日历时可请求微信订阅消息授权。

当前不使用：

- 手机号（微信授权）。
- 地理位置。
- 手机系统通讯录。
- 微信/手机系统日历写入权限。
- 相册、摄像头、麦克风。
- 广告追踪或第三方统计 SDK。

## 后台勾选建议

| 后台选项                         | 是否勾选 | 原因                                                       |
| -------------------------------- | -------- | ---------------------------------------------------------- |
| 用户信息（微信昵称、头像）       | 勾选     | 使用 `open-type="chooseAvatar"` 和 `input type="nickname"` |
| 自定义                           | 勾选     | 提醒、日历、应用内联系人、CLI 令牌、订阅消息等             |
| 通讯录（仅写入）权限             | 不勾选   | 不写入手机/微信系统通讯录                                  |
| 日历（仅写入）权限               | 不勾选   | 不写入手机/微信系统日历                                    |
| 地址                             | 不勾选   | 无收货/通讯地址                                            |
| 位置信息 / 选择的位置信息        | 不勾选   | 日历「地点」为用户手填文本                                 |
| 手机号                           | 不勾选   | 不调用 `getPhoneNumber`                                    |
| 选中的照片或视频 / 相册 / 摄像头 | 不勾选   | 不选择或上传媒体                                           |
| 麦克风                           | 不勾选   | 不录音                                                     |
| 设备信息、传感器、蓝牙           | 不勾选   | 未使用                                                     |
| 操作日志                         | 暂不勾选 | 审核要求时再补充                                           |

最小推荐勾选：`用户信息（微信昵称、头像）` + `自定义`（多条，见 **2026-06-07** 章节）。

## 数据存储和共享

| 数据                       | 存储位置                                                 | 保留策略                                 | 对外共享 |
| -------------------------- | -------------------------------------------------------- | ---------------------------------------- | -------- |
| 微信 `openid` / `unionid`  | 服务端数据库 `identities`                                | 随账号保留；删除账号能力上线后应一并删除 | 不共享   |
| 用户昵称、头像 URL         | 服务端数据库 `users`                                     | 随账号保留；用户后续可修改               | 不共享   |
| 提醒、日历、应用内联系人   | 服务端数据库                                             | 用户主动删除后按业务删除或软删除策略处理 | 不共享   |
| CLI 访问令牌               | 服务端只保存 token hash；明文仅创建时返回一次            | 用户可主动吊销；过期后不可用             | 不共享   |
| 剪贴板文本                 | 用户主动粘贴到输入框后作为表单内容；创建后按业务数据保存 | 未创建前不保存                           | 不共享   |
| 小程序本地 session token   | 微信小程序本地存储                                       | 用户退出登录后清除                       | 不共享   |
| 订阅消息授权结果和通知日志 | 服务端数据库                                             | 用于触达审计和错误排查                   | 不共享   |

当前项目未接入广告 SDK、第三方统计 SDK 或第三方数据处理方。生产 API 域名为 `https://xingxiaolin.cn`。

## 代码侧隐私授权实现

- `app.json` 启用 `__usePrivacyCheck__`。
- `app.ts` 监听 `wx.onNeedPrivacyAuthorization`。
- 用户点击微信登录后，先调用 `wx.requirePrivacyAuthorize`；同意后才继续 `wx.login`。
- 自定义弹窗组件使用 `open-type="agreePrivacyAuthorization"`。
- 用户可通过 `wx.openPrivacyContract` 查看隐私保护指引。

涉及隐私授权的页面：「我的」页首次登录头像昵称确认；「个人资料」页修改头像昵称。

## 不采集或不使用

- 手机号：不使用 `getPhoneNumber`。
- 手机系统通讯录：不读取、不写入。
- 微信/手机系统日历：不读取、不写入。
- 地理位置：不使用 `getLocation` / `chooseLocation`。
- 相册、摄像头、媒体文件：不使用 `chooseImage` / `chooseMedia`。
- 录音：不使用录音接口。
- 剪贴板读取：不调用 `getClipboardData`。
- 广告追踪：不接入广告或第三方追踪 SDK。

## 开发门禁

任何涉及以下能力的变更都必须先更新本文（新增日期版本），并在微信公众平台完成对应报备后再提审：

- 头像上传到服务端或对象存储。
- 新增手机号、邮箱等账户资料字段。
- 读取或写入手机系统通讯录。
- 读取或写入微信/手机系统日历。
- 使用定位或位置选择。
- 上传图片、文件、语音或视频。
- 新增 `getClipboardData` 或自动/后台读取剪贴板。
- 接入广告、统计或第三方 SDK。

发布前固定检查：

```bash
rg "getPhoneNumber|chooseImage|chooseMedia|getLocation|chooseLocation|getClipboardData|getUserProfile|requestSubscribeMessage|chooseAvatar|type=\"nickname\"|onNeedPrivacyAuthorization|requirePrivacyAuthorize|agreePrivacyAuthorization" apps/miniapp/miniprogram apps/miniapp/types
```

检查结果须与本文数据清单及**当前日期版本**的可复制文案一致。
