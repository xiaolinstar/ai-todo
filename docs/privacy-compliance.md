# ai-todo 微信小程序隐私报备清单

日期：2026-06-02

本文用于微信公众平台“小程序隐私保护指引”配置。重点是对齐后台弹窗里的“使用用户信息类型”，避免把本应用内部数据误报成微信/手机系统权限。

本文不是法律意见。后台选项名称可能随微信公众平台调整，最终以微信公众平台当前页面为准。

## 当前结论

`ai-todo` 当前是个人提醒、日历和联系人管理工具。小程序实际使用的微信隐私相关能力较少：

- 使用微信登录凭证完成账号登录。
- 用户可主动选择微信头像和昵称作为小程序账户资料。
- 用户可创建提醒、日历和本应用内联系人。
- 用户可创建 CLI 访问令牌，并主动复制到剪贴板。
- 如配置订阅消息模板，创建提醒时可请求微信订阅消息授权。

当前不使用：

- 手机号。
- 地理位置。
- 手机系统通讯录。
- 微信/手机系统日历写入权限。
- 相册、摄像头、麦克风。
- 剪贴板读取。
- 广告追踪或第三方统计 SDK。

## 后台勾选建议

按你截图中的“使用用户信息类型”弹窗，建议这样填写：

| 后台选项 | 是否勾选 | 原因 |
|----------|----------|------|
| 用户信息（微信昵称、头像） | 勾选 | 当前使用 `open-type="chooseAvatar"` 和 `input type="nickname"`，用于账户头像昵称展示 |
| 自定义 | 建议勾选 | 提醒、日历、本应用内联系人、CLI 令牌名称等不属于截图中的标准系统权限，需要用自定义说明 |
| 通讯录（仅写入）权限 | 不勾选 | 当前不写入手机/微信系统通讯录，只维护 ai-todo 应用内联系人 |
| 日历（仅写入）权限 | 不勾选 | 当前不写入手机/微信系统日历，只维护 ai-todo 应用内日历 |
| 地址 | 不勾选 | 当前没有收集用户收货地址或通讯地址 |
| 位置信息 / 选择的位置信息 | 不勾选 | 当前不调用定位或位置选择接口；日历“地点”只是用户手动输入文本 |
| 手机号 | 不勾选 | 当前不调用 `getPhoneNumber` |
| 选中的照片或视频 / 相册权限 / 摄像头 | 不勾选 | 当前不选择或上传图片、视频，不调用相册或摄像头 |
| 麦克风 | 不勾选 | 当前不录音 |
| 设备信息、传感器、蓝牙 | 不勾选 | 当前不使用这些能力 |
| 操作日志 | 暂不勾选 | 服务端有命令审计表，但当前小程序隐私报备建议先按用户主动提交的数据报备；如后台审核要求，可再补充“操作日志” |

最小推荐勾选：

```text
用户信息（微信昵称、头像）
自定义
```

## 后台填写文案

微信后台常见句式是“开发者收集你的 xx，用途是 yy”。下面按这个口径给出可直接复制的文案。

### 用户信息（微信昵称、头像）

```text
开发者收集你的微信头像、昵称，用途是在“我的”页面展示小程序账户资料，帮助你识别当前登录账号。
```

### 自定义

后台按多条自定义信息填写，按下面逐条新增。

#### 账号标识

```text
开发者收集你的微信账号标识，用途是完成微信登录并绑定小程序内部用户账号，识别用户身份，并隔离不同用户的提醒、日历、应用内联系人和 CLI 访问数据。
```

备注：账号标识主要指服务端通过微信登录凭证获取的 `openid`；如微信返回 `unionid`，也会保存。系统会生成独立的内部用户 ID 管理业务数据，不会将微信 `openid` 作为提醒、日历、联系人等业务数据主键。

#### 提醒

```text
开发者收集你的提醒，用途是创建、展示和管理你的个人提醒。
```

备注：提醒包括标题、备注、截止时间、提醒时间和关联的应用内联系人。

#### 日历

```text
开发者收集你的日历，用途是创建、展示和管理你的个人日历。
```

备注：日历包括标题、描述、开始时间、结束时间、地点文本和关联的应用内联系人。地点文本由用户手动输入，不属于定位信息。

#### 应用内联系人

```text
开发者收集你的应用内联系人，用途是创建、展示和管理你的 ai-todo 应用内联系人，并将联系人关联到提醒和日历。
```

备注：应用内联系人包括姓名、昵称、公司、职位、备注，以及邮箱、手机号、微信号或其他联系方式。当前不读取、不写入手机系统通讯录。

#### CLI 访问令牌

```text
开发者收集你的 CLI 访问令牌，用途是让你在本机 CLI 或 Agent 中访问同一账号下的提醒、日历和应用内联系人数据。
```

备注：服务端只保存令牌 hash，明文令牌仅创建时返回一次；用户可主动吊销令牌。

#### 订阅消息授权

```text
开发者收集你的订阅消息授权，用途是在你创建提醒后，按你的授权发送提醒通知。
```

备注：仅在配置微信订阅消息模板并由用户主动授权后使用。

#### 剪贴板

如果微信后台出现“开发者读取你的剪切板，用途是”且该项为必填，建议按实际能力这样写：

```text
开发者收集你的剪切板文本，用途是在你主动点击“从剪切板导入”或相关复制/导入操作时，辅助创建提醒、日历或应用内联系人，或复制 CLI 访问令牌。小程序不会自动读取剪切板，也不会在未经你确认的情况下保存剪切板内容。
```

## 数据存储和共享

| 数据 | 存储位置 | 保留策略 | 对外共享 |
|------|----------|----------|----------|
| 微信 `openid` / `unionid` | 服务端数据库 `identities` | 随账号保留；删除账号能力上线后应一并删除 | 不共享 |
| 用户昵称、头像 URL | 服务端数据库 `users` | 随账号保留；用户后续可修改 | 不共享 |
| 提醒、日历、应用内联系人 | 服务端数据库 | 用户主动删除后按业务删除或软删除策略处理 | 不共享 |
| CLI 访问令牌 | 服务端只保存 token hash；明文仅创建时返回一次 | 用户可主动吊销；过期后不可用 | 不共享 |
| 小程序本地 session token | 微信小程序本地存储 | 用户退出登录后清除 | 不共享 |
| 订阅消息授权结果和通知日志 | 服务端数据库 | 用于提醒触达审计和错误排查 | 不共享 |

当前项目未接入广告 SDK、第三方统计 SDK 或第三方数据处理方。生产 API 域名为 `https://wodi.games`。

## 代码侧隐私授权实现

当前小程序已接入微信隐私授权提示：

- `app.json` 启用 `__usePrivacyCheck__`。
- `app.ts` 监听 `wx.onNeedPrivacyAuthorization`。
- 用户点击微信登录后，先调用 `wx.requirePrivacyAuthorize`；同意后才继续 `wx.login`。
- 自定义弹窗组件使用 `open-type="agreePrivacyAuthorization"`。
- 用户可通过 `wx.openPrivacyContract` 查看隐私保护指引。

涉及隐私授权的页面：

- “我的”页首次登录头像昵称确认。
- “个人资料”页修改头像昵称。

## 不采集或不使用

当前版本不采集、不调用、不读取以下能力：

- 手机号：不使用 `getPhoneNumber`。
- 手机系统通讯录：不读取、不写入。
- 微信/手机系统日历：不读取、不写入。
- 地理位置：不使用 `getLocation` / `chooseLocation`。
- 相册、摄像头、媒体文件：不使用 `chooseImage` / `chooseMedia`。
- 录音：不使用录音接口。
- 剪贴板读取：不使用 `getClipboardData`。
- 广告追踪：不接入广告或第三方追踪 SDK。

## 0.1.2 开发门禁

任何涉及以下能力的变更都必须先更新本文，并在微信公众平台完成对应报备后再提审：

- 头像上传到服务端或对象存储。
- 新增手机号、邮箱等账户资料字段。
- 读取或写入手机系统通讯录。
- 读取或写入微信/手机系统日历。
- 使用定位或位置选择。
- 上传图片、文件、语音或视频。
- 读取剪贴板。
- 接入广告、统计或第三方 SDK。

发布前固定检查：

```bash
rg "getPhoneNumber|chooseImage|chooseMedia|getLocation|chooseLocation|getClipboardData|getUserProfile|requestSubscribeMessage|chooseAvatar|type=\"nickname\"|onNeedPrivacyAuthorization|requirePrivacyAuthorize|agreePrivacyAuthorization" apps/miniapp/miniprogram apps/miniapp/types
```

检查结果需要和本文的数据清单一致；不一致时先更新报备，再发布版本。
