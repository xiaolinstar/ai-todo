# ai-todo 微信小程序隐私报备清单

日期：2026-06-01

本文用于微信小程序后台隐私保护指引、用户隐私相关接口报备，以及 `0.1.2` 后续开发前的合规检查。平台后台的具体类目名称可能会随微信公众平台调整，最终填写时以微信公众平台页面展示为准；本文以当前代码实际采集、存储和使用的数据为准。

## 当前结论

`ai-todo` 当前属于轻量个人效率工具，主要处理用户主动创建的提醒、日程和联系人数据。小程序端没有读取手机通讯录、手机号、地理位置、相册、摄像头、录音，也没有读取剪贴板内容。

当前需要重点报备或在隐私指引中说明的内容：

- 微信用户标识：用于登录、账号识别和数据隔离。
- 用户头像、昵称：用于“我的”页账户展示，由用户主动确认。
- 提醒、日程、联系人内容：由用户主动填写，用于提供待办提醒、日程和联系人管理。
- 订阅消息授权结果：用于提醒触达；当前真实投递不是 `0.1.1` 发布门槛，但相关接口已存在。
- 剪贴板写入：仅用于用户主动复制 CLI 访问令牌，不读取剪贴板。

## 微信后台报备建议

在微信公众平台填写隐私保护指引时，建议按“最小必要”原则报备当前版本实际能力。

| 建议报备项 | 当前使用场景 | 是否必填 |
|------------|--------------|----------|
| 用户头像 | 首次微信登录后，用户可选择微信头像作为小程序账户头像 | 是 |
| 用户昵称 | 首次微信登录后，用户可选择/填写昵称作为小程序账户昵称 | 是 |
| 用户标识信息 | 通过 `wx.login` 换取 `openid`，用于账号登录和用户数据隔离；如微信返回 `unionid` 也会保存 | 是 |
| 订阅消息 | 创建提醒后，如配置模板 ID，可请求用户授权提醒通知 | 如果启用提醒触达则必填 |
| 联系人信息 | 用户手动录入联系人姓名、昵称、公司、职位、备注和联系方式，用于关联提醒/日程 | 是 |
| 提醒和日程内容 | 用户手动录入标题、备注、时间、地点文本等，用于提醒和日程管理 | 是 |
| 剪贴板 | 用户主动复制 CLI 访问令牌到剪贴板；不读取剪贴板内容 | 如后台要求声明剪贴板能力则填写 |

不要报备当前没有使用的能力，例如手机号、地理位置、手机通讯录、相册、摄像头、录音等。后续如果新增这些能力，必须先更新本文和微信后台报备，再发布小程序版本。

## 可直接填写的用途说明

### 用户头像

使用目的：用于在小程序“我的”页面展示用户账户头像，帮助用户识别当前登录账号。

使用场景：用户首次微信登录后，可主动选择微信头像作为小程序账户头像；后续资料设置页上线后，用户可再次修改。

### 用户昵称

使用目的：用于在小程序“我的”页面展示用户账户昵称，帮助用户识别当前登录账号。

使用场景：用户首次微信登录后，可主动选择或填写昵称作为小程序账户昵称；后续资料设置页上线后，用户可再次修改。

### 用户标识信息

使用目的：用于完成微信登录、识别用户身份、隔离不同用户的提醒、日程、联系人和 CLI 访问数据。

使用场景：用户点击“微信登录”后，小程序调用 `wx.login` 获取临时登录凭证，服务端通过微信接口换取 `openid`；如微信返回 `unionid`，服务端也会保存用于同一主体身份识别。

### 提醒内容

使用目的：用于创建、展示、更新、完成和删除用户的个人提醒。

使用场景：用户在小程序或 CLI 中主动填写提醒标题、备注、截止时间、提醒时间和关联联系人。

### 日程内容

使用目的：用于创建、展示和管理用户的个人日程。

使用场景：用户在小程序或 CLI 中主动填写日程标题、描述、开始时间、结束时间、地点文本和关联联系人。当前地点字段是用户手动输入的文本，不使用手机定位能力。

### 联系人信息

使用目的：用于用户维护个人联系人，并将联系人关联到提醒和日程。

使用场景：用户在小程序或 CLI 中主动添加联系人姓名、昵称、公司、职位、备注，以及邮箱、手机号、微信号或其他联系方式。当前不会读取手机系统通讯录。

### 订阅消息授权

使用目的：用于在用户创建提醒后，按用户授权发送提醒通知。

使用场景：用户创建提醒时，如后端配置了微信订阅消息模板 ID，小程序会调用 `wx.requestSubscribeMessage` 请求授权；用户可以同意或拒绝。

### 剪贴板写入

使用目的：方便用户把自己创建的 CLI 访问令牌复制到本机剪贴板，用于配置本地 CLI 或 Agent。

使用场景：用户在“我的”页主动创建 CLI 访问令牌后，点击复制。小程序只写入剪贴板，不读取剪贴板内容。

## 数据存储和共享

| 数据 | 存储位置 | 保留策略 | 对外共享 |
|------|----------|----------|----------|
| 微信 `openid` / `unionid` | 服务端数据库 `identities` | 随账号保留；删除账号能力上线后应一并删除 | 不共享 |
| 用户昵称、头像 URL | 服务端数据库 `users` | 随账号保留；用户后续可修改 | 不共享 |
| 提醒、日程、联系人 | 服务端数据库 | 用户主动删除后按业务删除或软删除策略处理 | 不共享 |
| CLI 访问令牌 | 服务端只保存 token hash；明文仅创建时返回一次 | 用户可主动吊销；过期后不可用 | 不共享 |
| 小程序本地 session token | 微信小程序本地存储 | 用户退出登录后清除 | 不共享 |
| 订阅消息授权结果和通知日志 | 服务端数据库 | 用于提醒触达审计和错误排查 | 不共享 |

当前项目未接入广告 SDK、第三方统计 SDK 或第三方数据处理方。生产 API 域名为 `https://wodi.games`。

## 不采集或不使用

当前版本不采集、不调用、不读取以下能力：

- 手机号：不使用 `getPhoneNumber`。
- 手机通讯录：不读取系统通讯录。
- 地理位置：不使用 `getLocation`。
- 相册、摄像头、媒体文件：不使用 `chooseImage` / `chooseMedia`。
- 录音：不使用录音接口。
- 剪贴板读取：不使用 `getClipboardData`。
- 广告追踪：不接入广告或第三方追踪 SDK。

## 0.1.2 开发前合规门禁

开始 `0.1.2` 开发时，任何涉及以下能力的变更都必须先更新本文，并在微信公众平台完成对应报备后再提审：

- 头像上传到服务端或对象存储：需要确认头像文件的存储、访问控制、删除策略和隐私说明。
- 个人资料设置页：如果仍只修改头像和昵称，不新增隐私类目；如果新增手机号、邮箱等，需要补充报备。
- 真实提醒触达 worker：需要确认订阅消息模板 ID、模板字段、触达场景和用户授权文案。
- 联系人导入：如果读取手机通讯录，必须新增通讯录能力报备，并重新评估产品必要性。
- 地点增强：如果从手动地点文本升级为定位能力，必须新增地理位置报备。
- 图片、附件、语音：如果支持上传图片、文件或录音，必须新增对应类目报备。

合规检查建议作为每个版本发布前固定步骤：

```bash
rg "getPhoneNumber|chooseImage|chooseMedia|getLocation|getClipboardData|getUserProfile|requestSubscribeMessage|chooseAvatar|type=\"nickname\"" apps/miniapp/miniprogram
```

检查结果需要和本文的数据清单一致；不一致时先更新报备，再发布版本。
